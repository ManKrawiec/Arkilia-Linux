# Code of Conduct

## Our Pledge

Arkilia Linux should be a friendly and practical place to collaborate. We want people to be able to ask questions, report issues, suggest changes, and learn without being mocked or pushed away.

## Expected Behavior

- Be respectful and patient.
- Assume good intent, especially with new contributors.
- Keep criticism focused on the work, not the person.
- Explain technical feedback clearly.
- Make room for people who are still learning.

## Unacceptable Behavior

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Trolling or deliberately disruptive comments.
- Sharing private information without permission.
- Repeatedly ignoring maintainer requests.

## Scope

This code of conduct applies to the Arkilia Linux repository, issues, pull requests, discussions, and any related project spaces.

## Enforcement

Project maintainers may remove comments, close issues, reject pull requests, or block users when behavior makes the project harder or less safe to work on.

If you see a problem, open an issue or contact a maintainer through GitHub.
